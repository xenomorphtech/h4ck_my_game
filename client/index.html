Exploit recap

Statements

send Packet { field: value }
Emit one client packet at the current virtual time.
sleep ms
Advance virtual time. Not allowed inside send_batch or at blocks.
send_batch { Packet { field: value } ... }
Sends a batch of packets on the same packet frame. Packet lines inside the block omit send.
at(ms) { ... }
Emit nested packets at an absolute virtual timestamp.
for i in start..end { ... }
Repeat the block with an integer range; end is exclusive.

Values

Fields accept integers, negative integers, booleans, quoted strings, and bare words. Arrays and nested objects are not parsed.

# starts a comment. Each statement and each brace must be on its own line.

Clarifications

  • send_batch { must open the packet-frame batch; close it with a line containing only }.
  • Loop variables substitute only when the whole field value is the variable, such as x: i.
  • await Packet { ... } and let x = await Packet { ... } are accepted placeholders; they do not bind packet fields yet.
  • Packet names and field names are scenario-specific. Use the Packets tab for the current schema.

Examples

send Attack { target: 1, power: 40 }
sleep 750
send Attack { target: 1 }
send_batch {
  Attack { target: 1 }
  Attack { target: 1 }
  Attack { target: 1 }
}
for x in 1..8 {
  send Move { x: x, y: 3 }
  sleep 20
}
at(0) {
  send Buyout { listing: 21, quantity: 1 }
  send Buyout { listing: 21, quantity: 1 }
}
